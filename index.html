Los geht's...

Neues Repository erstellen


git init
erstelle ein neues Verzeichnis, öffne es und führe
git init
aus, um ein neues git-Repository anzulegen.

Repository auschecken


git clone benutzername@host:/pfad/zum/repository
geht auch lokal mit git clone /pfad/zum/repository

add & commit


						git add
						git add *
						git add .

						git commit -m "Commit-Nachricht"
Nach commit befinden sich die Änderungen im HEAD, aber nicht im entfernten Repository

Änderungen hochladen


						git push
						git push origin master

						git remote add origin < server >
Hinzufügen eines Repos zu lokalem Verzeichnis

Branching


						git checkout -b feature_x
						git checkout master
						git branch -d feature_x
						git push origin < branch >
- master-Branch ist Standard
- Zusammenführen = mergen
- Branch erzeugen und dahin wechseln: git checkout -b feature_xy
- zurück zu Master: git checkout master
- Branch löschen: git branch -d feature_xy
- ins Repo hochladen: git push origin feature_xy

update & merge


						git pull
						git merge < branch >
						git add < dateiname >
						git diff < quell_branch > < ziel_branch >
- master-Branch ist Standard
- andere Branches kann man anlegen und später wieder zusammenführen (mergen)

Logs anschauen


						git log

Änderungen zurücknehmen


						git checkout -- < filename >

						git fetch origin
						git reset --hard origin/master
(1) git checkout -- < filename >: Zurücksetzen auf den letzten Stand von HEAD
(2) Alle Änderungen zurücknehmen: vom entfernten Repo holen:
